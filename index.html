CRUD OPERATIONS

  • Select from a Table

(

  • Update in a Table
-- UPDATE TABLE

UPDATE TABLE_NAME
SET COLUMN_NAME
WHERE CONDITION
  • Delete Table
-- DELETE TABLE

DELETE FROM TABLE_NAME
WHERE CONDITION
  • Drop Table
-- DROP TABLE
DROP TABLE TABLE_NAME
  • Truncate Table
-- TRUNCATE TABLE
TRUNCATE TABLE TABLE_NAME
  • Delete is used to remove single or multiple rows from the Table.
  • Truncate removes all the rows but keeping the Table Schema.
  • Drop deletes the Table Data as well as Table Schema.

Sub Query

In SQL a Subquery can be simply defined as a query within another query. In other words we can say that a Subquery is a query that is embedded in WHERE clause of another SQL query.

  • ORDER BY command cannot be used in a Subquery. GROUPBY command can be used to perform same function as ORDER BY command.
  • A subquery can have only one column in the SELECT clause, unless multiple columns are in the main query for the subquery to compare its selected columns.
SELECT * FROM TABLE_NAME
WHERE COLUMN_NAME = (SELECT COLUMN_NAME FROM TABLE_NAME2)

Basic Terminologies

  1. And / Or : The AND and OR operators are used to filter records based on more than one condition:
  • The AND operator displays a record if all the conditions separated by AND are TRUE.
  • The OR operator displays a record if any of the conditions separated by OR is TRUE.
-- And
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE CONDITION1 AND CONDITION2

-- Or
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE CONDITION1 OR CONDITION2
  1. NOT : The NOT operator displays a record if the condition(s) is NOT TRUE.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE NOT CONDITION
  1. Where : The WHERE clause is used to filter records.It is used to extract only those records that fulfill a specified condition.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE CONDITION
  1. Order By : The ORDER BY keyword sorts the records in ascending order by default. To sort the records in descending order, use the DESC keyword.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
ORDER BY COLUMN_NAME1, COLUMN_NAME2 ASC|DESC
  1. Group By : The GROUP BY statement is often used with aggregate functions to group the result-set by one or more columns.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE CONDITION
GROUP BY COLUMN_NAME1, COLUMN_NAME2
ORDER BY COLUMN_NAME1, COLUMN_NAME2 ASC|DESC

-- or if you want to Sum COLUMN_NAME2

SELECT COLUMN_NAME1, SUM(COLUMN_NAME2)
FROM TABLE_NAME
WHERE CONDITION
GROUP BY COLUMN_NAME1
ORDER BY COLUMN_NAME1, COLUMN_NAME2 ASC|DESC
  1. Having : The HAVING clause was added to SQL because the WHERE keyword cannot be used with aggregate functions. The HAVING statement is often used as a where statement after GROUP BY.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE CONDITION
GROUP BY COLUMN_NAME1, COLUMN_NAME2
HAVING CONDITION
ORDER BY COLUMN_NAME1, COLUMN_NAME2 ASC|DESC
  1. Exists : The EXISTS operator is used to test for the existence of any record in a subquery.The EXISTS operator returns TRUE if the subquery returns one or more records.
IF EXISTS (SELECT COLUMN_NAME FROM TABLE_NAME WHERE CONDITION)
BEGIN
-- If the record dos not exist then it will enter this block.
END
  1. Not Exists : The NOT EXISTS operator returns TRUE if the record does not exist
IF NOT EXISTS (SELECT COLUMN_NAME FROM TABLE_NAME WHERE CONDITION)
BEGIN
-- If the record dos not exist then it will enter this block.
END
  1. IS NULL / IS NOT NULL : It can be used to check if value in column is Null or not.
SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE COLUMN_NAME1 IS NULL

-- or

SELECT COLUMN_NAME1, COLUMN_NAME2
FROM TABLE_NAME
WHERE COLUMN_NAME1 IS NOT NULL
  1. ISNULL : If you know the value is NULL and you want it to replace with a specific value.
ISNULL(COLUMN_NAME,'')

JOINS

A JOIN clause is used to combine rows from two or more tables, based on a related column between them.

SELECT Orders.OrderID, Customers.CustomerName, Orders.OrderDate
FROM Orders
INNER JOIN Customers WITH (NOLOCK) ON Orders.CustomerID=Customers.CustomerID;

-- After on there will be the condition on which you want to join.

Here are the different types of the JOINs in SQL

  • (INNER) JOIN: Returns records that have matching values in both tables.
  • LEFT (OUTER) JOIN: Returns all records from the left table, and the matched records from the right table.
  • RIGHT (OUTER) JOIN: Returns all records from the right table, and the matched records from the left table.
  • FULL (OUTER) JOIN: Returns all records when there is a match in either left or right table.

CASE

The CASE expression goes through conditions and returns a value when the first condition is met (like an if-then-else statement). So, once a condition is true, it will stop reading and return the result. If no conditions are true, it returns the value in the ELSE clause. If there is no ELSE part and no conditions are true, it returns NULL.

CASE
    WHEN CONDITION1 THEN RESULT1
    WHEN CONDITION2 THEN RESULT2
    WHEN CONDITIONN THEN RESULTN
    ELSE RESULT
END

Ternary Operator

The IIF() function returns a value if a condition is TRUE, or another value if a condition is FALSE.

SELECT IIF(500<1000, 'YES', 'NO');

Cast and Convert

CAST : The CAST() function converts a value (of any type) into a specified datatype.

SELECT CAST(25.65 AS int)

SELECT CAST('2017-08-25' AS datetime)

CONVERT : The CONVERT() function converts a value (of any type) into a specified datatype.

SELECT CONVERT(int, 25.65)

SELECT CONVERT(datetime, '2017-08-25')
